Import data¶

to import data, we use an instance of the DataLoaders class:

[7]:

from pathlib import Path
import sys
sys.path.append('../')
from ParticlePhaseSpace import DataLoaders
from ParticlePhaseSpace.ParticlePhaseSpace import ParticlePhaseSpace

test_data_loc = Path(r'../examples/coll_PhaseSpace_xAng_0.00_yAng_0.00_angular_error_0.0.phsp').absolute()
ps_data = DataLoaders.LoadTopasData(test_data_loc)

Once we have a DataLoader instance, we can pass it to ParticlePhaseSpace for analysis.3 Note that while we are using topas data in this example, any DataLoader will guarantee data that can be passed to ParticlePhaseSpace. For a full list of current DataLoaders, see HERE. For notes on writing new DataLoaders, see HERE.

[8]:

PS = ParticlePhaseSpace(ps_data)

Seperating, adding, and subtracting phase space objects¶

You can easily add, subtract, and seperate phase space objects:

[13]:

electron_PS = PS('electrons')  # this generates a phase space with only the electrons
# you can also use pfg codes:
electron_PS = PS(11)  # this is the same
electron_PS.plot_energy_histogram()

particle type check not implemented

You can also pass a list of particles to get multiple phase spaces at once:

[14]:

gamma_PS, positron_PS = PS(['gammas', 'positrons'])  # you can also pass a list

You can subtract one phase space from another. For example, the following will produce a new phase space object where all the gamma particles are removed:

[15]:

no_gamma_PS = PS - gamma_PS
no_gamma_PS.plot_energy_histogram()

particle type check not implemented

You can also add phase spaces together:

[16]:

Original_PS = no_gamma_PS + gamma_PS
Original_PS.plot_energy_histogram()

particle type check not implemented

However: you cannot add identical particles into an existing phase space. If you really want to do this, you need to update the particle_ID field first:

[17]:

this_wont_work = electron_PS + electron_PS
